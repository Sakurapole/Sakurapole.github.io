LaityHTool项目简介

LaityHTool是一个我自己想到什么功能就会加在上面的项目，运行于整个PC平台，预计会开发的功能大概有：

  1. Todo列表（实现云同步） √
  2. BDownloader（B站下载器）
    • 收藏夹下载 √
    • AV/BV号下载
    • 链接下载
  3. Spyder Center（服务器爬虫总控，监控服务器上和本机爬虫运行情况）
  4. Note Reader（文本阅读器，支持普通笔记以及markdown文本）
    …

技术简介

采用Electron-vue前端技术，Electron是PC上跨平台的集成了V8+nodejs的采用前端开发语言的框架。Vue是前端的流行开发框架之一。

本项目的后端开发语言不定，目前还未进行到后端

项目前端问题

记录项目中查询过资料的前端问题

初始化项目时需要更新官方模板中Electron和Node的版本

命令行执行
yarn upgrade-interactive –latest

选择electron、electron-builder 、electron-debug、electron-devtools-installer这四项更新。然后修改electron主进程文件index.js中的创建主窗口代码为：

mainWindow = new BrowserWindow({
  height: xxx,
  useContentSize: true,
  width: xxx,
  webPreferences: {
    nodeIntegration: true, //在网页中集成Node，electron高版本默认关闭
  }
})

Electron配置请求头的Referer

由于请求B站下的资源需要请求中携带B站的Referer，而Referer是没有办法直接在axios中设置的，会抛出unsafe-header，因为浏览器不允许修改Referer，所以我们直接修改electorn即可，因为electron是提供浏览器环境的。在electron的主进程index.js中设置一下代码

const bilibiliFilter = { // 当请求这些链接时需要携带Referer
  urls: ["https://*.bilibili.com/*", "http://*.bilibili.com/*", "http://*.hdslb.com/*", "https://*.bilivideo.com/*", "https://*/*"]
}

// electron的session模块提供了在发送请求前修改请求头的方法 使用前需要导入session：const { session } = require('electron')
session.defaultSession.webRequest.onBeforeSendHeaders(bilibiliFilter, (details, callback) => {
  details.requestHeaders['Referer'] = 'http://www.bilibili.com'
  callback({ requestHeaders: details.requestHeaders });
})
